## Submit Interview Session

API để nộp toàn bộ bài phỏng vấn sau khi candidate hoàn thành trả lời các câu hỏi.

## Endpoint
`POST /api/interviews/{id}/submit`

## Authentication
Required (JWT Token) - Candidate

## Path Parameters
- `id` (integer, required): ID của interview session.

## Request Body
Không có body. FE chỉ cần gửi request submit với path param và token.

## Business Rules
- **Existing session**: Nếu session không tồn tại → `"Interview not found."`.
- **Ownership**: User hiện tại phải là owner của session, nếu không → `"You do not have permission to submit this interview."`.
- **Status hợp lệ**:
  - Domain chỉ cho phép submit nếu status hiện tại là `in_progress`.
  - Nếu khác, domain ném lỗi `"Interview session can only be submitted if status is in_progress."`.
- **Trạng thái sau khi submit**:
  - Domain chuyển status sang `submitted`.
  - Set `submitted_at` là thời gian hiện tại.
- **AI Evaluation**:
  - Sau khi submit, hệ thống dispatch job `RunAiEvaluationJob` để chạy đánh giá AI cho toàn bộ bài phỏng vấn.

## Response Success (200 OK)
```json
{
  "success": true,
  "data": {
    "interview_session_id": 14,
    "status": "submitted",
    "submitted_at": "2026-01-28 11:30:00"
  },
  "message": "Interview session submitted successfully."
}
```

## Error Cases
- **400 Bad Request**:
  - `"Interview not found."`
  - `"You do not have permission to submit this interview."`
  - `"Interview session can only be submitted if status is in_progress."`
- **401 Unauthorized**: Thiếu hoặc token không hợp lệ.
- **403 Forbidden**: Session không thuộc về user hiện tại.

## cURL Example
```bash
curl -X POST "http://localhost:8000/api/interviews/14/submit" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Accept: application/json"
```
