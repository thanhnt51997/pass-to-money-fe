# List Interview Sessions

API để lấy danh sách các phiên phỏng vấn của candidate hiện tại.

## Endpoint
`GET /api/interviews`

## Authentication
Required (JWT Token) - Candidate

## Query Parameters
- `status`: (optional) Filter theo status (created, started, in_progress, submitted, evaluated)
- `stack`: (optional) Filter theo stack (Backend, Frontend, etc.)
- `level`: (optional) Filter theo level (Junior, Senior, etc.)
- `per_page`: (optional) Số items per page, default: 15

## Filter Examples

### Filter by Status
```
GET /api/interviews?status=evaluated
```

### Filter by Stack and Level
```
GET /api/interviews?stack=Backend&level=Junior
```

### Pagination
```
GET /api/interviews?per_page=10&page=2
```

## Response Fields
- `id`: ID của session
- `template_id`: ID của template được sử dụng
- `level`: Cấp độ (snapshot từ template)
- `stack`: Ngăn xếp công nghệ (snapshot từ template)
- `status`: Trạng thái hiện tại
- `started_at`: Thời gian bắt đầu
- `submitted_at`: Thời gian nộp bài
- `score_total`: Điểm tổng (chỉ có sau khi evaluated)

## Status Values
- `created`: Session vừa tạo, chưa bắt đầu
- `started`: Đã bắt đầu, đang xem câu hỏi
- `in_progress`: Đang làm bài
- `submitted`: Đã nộp bài, chờ đánh giá
- `evaluated`: Đã có kết quả đánh giá

## Sorting
Mặc định sort theo `created_at DESC` (mới nhất trước)

## Error Cases
- **401 Unauthorized**: Missing hoặc invalid token

## cURL Example
```bash
curl -X GET "http://localhost:8000/api/interviews?status=evaluated&per_page=10" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## Use Cases
- **Dashboard**: Hiển thị lịch sử phỏng vấn của candidate
- **Resume interview**: Tìm session đang `in_progress` để tiếp tục
- **View results**: Lọc sessions đã `evaluated` để xem kết quả
- **Statistics**: Theo dõi số lần phỏng vấn, tỷ lệ pass/fail

## Notes
- Chỉ hiển thị sessions của candidate hiện tại (auto-filter by user_id)
- Không hiển thị sessions của candidates khác
- Admin có thể dùng API riêng để xem tất cả sessions
