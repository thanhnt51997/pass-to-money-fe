# Module: Question Bank (Admin)

## 1. Domain Concepts
### Question Entity
| Field | Description |
|---|---|
| id | UUID |
| title | Short question title |
| content | Full question text |
| type | coding / theoretical / system_design / behavioral |
| stack | Backend / Frontend / Fullstack / DevOps / AI |
| technologies | Laravel, Java, NodeJS, React, etc |
| level | Fresher / Junior / Middle / Senior / Tech Lead |
| difficulty | Easy / Medium / Hard |
| expected_answer | Reference answer (admin only) |
| evaluation_criteria | Rubric used by AI |
| tags | Searchable keywords |
| status | draft / active / archived |
| created_by | Admin ID |
| created_at | Timestamp |
| updated_at | Timestamp |

## 2. API List & Document Indexing Rule
API docs must be grouped under `/docs/API/QuestionBank/`
File naming format: `{STT}.{ActionName}.md`

## 3. Global Rules
- Use RESTful conventions
- Follow OpenAPI 3.1
- Request/Response strictly JSON
- Use snake_case for fields
- Admin authentication required (JWT)
- Soft delete instead of hard delete
- Pagination required for list APIs
- Validation errors must be deterministic
- Error codes must be stable and documented
